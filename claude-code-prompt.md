# フリマ出品ストック — Claude Code 用プロンプト

## これは何か

メルカリ・PayPayフリマ（Yahoo!フリマ）・ラクマの3サイトに不用品を出品している個人ユーザーのための、**出品下書きストック＆再出品管理アプリ**。

フリマサイトには個人向けの出品APIが無いため、自動出品はしない。代わりに「出品する内容（写真・タイトル・説明文・価格）をこのアプリにストックしておき、各サイトの出品画面にコピペするだけの状態にする」ことで、出品と再出品の手間を最小化する。

利用者は1人。アカウント・サーバー・同期は不要。スマホのブラウザで使い、ホーム画面に追加してPWAとして使う。

## 技術スタック

- React + Vite + TypeScript
- データ保存: IndexedDB（`idb` ライブラリ使用）。写真は Blob のまま IndexedDB に保存する
- PWA: manifest + service worker（オフラインで開けること）
- 外部APIは一切使わない。完全ローカル完結

## データモデル

```ts
type Marketplace = 'mercari' | 'paypay' | 'rakuma';

type ListingStatus =
  | 'not_listed'   // 未出品
  | 'listed'       // 出品中
  | 'sold'         // 売却済み（このサイトで売れた）
  | 'take_down';   // 取り下げ推奨（他サイトで売れた）

interface Listing {
  status: ListingStatus;
  listedAt?: string;   // 出品した日（再出品バッジの起点）
  note?: string;       // サイト別メモ（任意）
}

interface Item {
  id: string;
  title: string;            // 商品タイトル
  description: string;      // 説明文（3サイト共通で1本）
  price: number;
  category?: string;        // 自由入力のカテゴリ名
  condition: string;        // 商品の状態（新品未使用/未使用に近い/目立った傷なし/やや傷汚れあり/傷汚れあり/全体的に状態が悪い）
  photos: Blob[];           // 最大10枚、1枚目がサムネイル
  listings: Record<Marketplace, Listing>;
  createdAt: string;
  updatedAt: string;
}

interface Snippet {
  id: string;
  label: string;   // 例: 「即購入OK」
  text: string;    // 挿入される本文
  order: number;
}

interface Settings {
  relistThresholdDays: number;  // 再出品バッジを出すまでの日数（デフォルト14日）
}
```

## 画面構成（下タブ4つ）

### 1. 📦 商品一覧（ホーム）

- 商品カードのリスト。カードには: サムネイル写真、タイトル、価格、3サイトそれぞれの状態バッジ（🔴メ / 🟡P / 🟢ラ のような小さいサイト別チップ＋状態の色）
- 状態の色分け: 未出品=グレー / 出品中=青 / 売却済み=緑 / 取り下げ推奨=赤
- フィルタチップ: すべて / 出品中 / 要再出品 / 売却済み / 未出品
- 「要再出品」= いずれかのサイトで出品中のまま `relistThresholdDays` 日以上経過している商品。カードに「⏰ そろそろ再出品」バッジを出す
- 「取り下げ推奨」がある商品はリスト最上部に出し、「⚠️ ほかのサイトで売れています。取り下げてください」と目立たせる（二重売れ防止が最優先）
- 右下にフローティング「＋」ボタン → 新規商品作成へ

### 2. ➕ 商品作成・編集

- 写真追加（カメラ/ライブラリから複数選択、並べ替え・削除可、1枚目がサムネイル）
- タイトル入力（40文字カウンター付き。メルカリのタイトル上限が40文字のため）
- 価格入力（3サイトの販売手数料の目安と手取り額を参考表示: メルカリ10%、PayPayフリマ5%、ラクマ4.5%〜。「目安です」と注記する）
- 商品の状態（6段階から選択）
- カテゴリ（自由入力＋過去に使ったカテゴリをサジェスト）
- **説明文エリア**:
  - 直接入力もできる
  - 「フォームから生成」ボタン → ブランド / サイズ / 色 / 購入時期 / 使用回数 / 傷・汚れの箇所 / 付属品 を入力すると、整った説明文テンプレを組み立てて挿入（ローカルのテンプレ処理。AIは使わない）
  - 「定型文を挿入」ボタン → 登録済みスニペットを選んで末尾に追加
- 保存すると3サイトとも「未出品」状態で商品一覧に入る

### 3. 📋 商品詳細（出品作業の中心画面）

この画面が本命。ここから各サイトへのコピペ出品を最速にする。

- 写真一覧（タップで拡大。長押しまたはボタンで端末に保存できるようにする — 各サイトのアプリに写真をアップし直すため）
- **コピーボタン群**: 「タイトルをコピー」「説明文をコピー」「価格をコピー」それぞれワンタップ。コピー成功時はトースト表示
- **サイト別ステータス**: メルカリ / PayPayフリマ / ラクマ の3行。各行に現在状態と操作ボタン
  - 「出品した」→ 出品中になり `listedAt` を記録
  - 「売れた！」→ このサイトが売却済みになり、**他サイトで出品中のものは自動で「取り下げ推奨」に変わる**
  - 「取り下げた」→ 未出品に戻る
  - 「再出品した」→ `listedAt` を今日に更新（再出品バッジが消える）
- 出品中のサイトには「出品からn日」を表示。閾値超過なら「⏰ そろそろ再出品」

### 4. ⚙️ 設定・定型文

- 定型文（スニペット）の一覧・追加・編集・削除・並べ替え
- 初期データとして数個入れておく（例: 「即購入OKです🙆」「値下げ交渉はコメントにてお願いします」「ペット・喫煙者おりません」「簡易梱包での発送となります」）
- 再出品バッジの日数設定（デフォルト14日）
- データの全削除（確認ダイアログ付き）

## 挙動の細かい仕様

- 説明文の自動生成テンプレは日本のフリマでよくある構成にする: 冒頭あいさつ → 商品名 → ブランド/サイズ/色 → 状態の説明 → 付属品 → 締めの定型文。空欄の項目は行ごとスキップして自然な文章になるようにする
- 「売れた！」を押したとき、他サイトが出品中なら「他のサイトの出品を取り下げてください」という確認モーダルを出す
- 写真は保存前にリサイズ（長辺1500px程度・JPEG圧縮）して容量を抑える
- クリップボードコピーは `navigator.clipboard.writeText` を使用

## デザイン

- モバイルファースト。幅440px程度の一列レイアウトをデスクトップでは中央寄せ
- 白の丸カード（角丸大きめ）＋やわらかい影、背景はごく薄い色
- フォント: M PLUS Rounded 1c
- 絵文字を積極的に使ったフレンドリーな見た目
- 下部固定のタブナビゲーション（📦 商品 / ➕ 追加 / ⚙️ 設定 の3タブ。商品詳細は一覧からの遷移）
- アプリ名: 「出品ストック」

## 作り方の注意

- 最初に動く最小構成（商品CRUD＋一覧＋詳細のコピー機能）を作ってから、説明文ジェネレーター・スニペット・再出品バッジを順に足していくこと
- 各ステップでビルドが通ることを確認してから次へ進むこと
