# 出品ストック 🏷️

メルカリ・PayPayフリマ（Yahoo!フリマ）・ラクマの3サイトに不用品を出品する人のための、**出品下書きストック＆再出品管理アプリ**。

フリマサイトには個人向けの出品APIが無いため、自動出品はしません。代わりに「出品する内容（写真・タイトル・説明文・価格）をこのアプリにストックしておき、各サイトの出品画面にコピペするだけの状態にする」ことで、出品と再出品の手間を最小化します。

## 主な機能

- 📦 **商品一覧**: サムネイル＋3サイトそれぞれの状態バッジ（未出品/出品中/売却済み/取り下げ推奨）。フィルタチップ付き
- ➕ **商品作成**: 写真（最大10枚・自動で長辺1500pxにリサイズ）、40文字カウンター付きタイトル、3サイトの手数料・手取り目安、説明文の自動生成（フォーム入力→テンプレ組み立て）、定型文挿入
- 📋 **商品詳細**: タイトル・説明文・価格のワンタップコピー、写真の拡大＆端末保存、サイト別ステータス管理（出品した/売れた！/取り下げた/再出品した）、「出品からn日」表示
- ⚠️ **二重売れ防止**: どこかのサイトで「売れた！」を押すと、他サイトの出品中は自動で「取り下げ推奨」になり警告表示
- ⏰ **再出品リマインド**: 出品から一定日数（デフォルト14日）で「そろそろ再出品」バッジ
- 📎 **定型文管理**: 追加・編集・削除・並べ替え

## 技術構成

- 単一 `index.html`（vanilla JS）+ `sw.js` + `manifest.json` のPWA
- データはすべて **IndexedDB** にローカル保存（写真はBlobのまま保存）。サーバー送信なし・アカウント不要
- オフラインで動作（Service Worker: HTMLはネットワーク優先、その他はキャッシュ優先）
- ホーム画面に追加してアプリとして使う想定

※元のプロンプトでは React + Vite + TypeScript 指定でしたが、このMacにNode.jsが無いため、同じ機能を単一HTML構成で実装しています（ヘルシー教アプリ・おこづかい帳と同じ構成）。

## ファイル構成

```
index.html          … アプリ本体（HTML/CSS/JSすべて）
sw.js               … Service Worker（オフライン対応）
manifest.json       … PWAマニフェスト
icon-192.png        … アプリアイコン
icon-512.png        … アプリアイコン（大）
apple-touch-icon.png … iOSホーム画面用アイコン
```

## スマホで使うには（GitHub Pages公開）

おこづかい帳と同じ手順です:

1. GitHubで新しいリポジトリを作る（例: `shuppin-stock`、Public）
2. このフォルダのファイルをアップロード
3. Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後に `https://<ユーザー名>.github.io/shuppin-stock/` で開ける
5. iPhoneのSafariで開いて「共有 → ホーム画面に追加」
